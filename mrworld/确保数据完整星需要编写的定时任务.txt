字段的变动场景和字段参与的场景
1.space :
CURD空间
name         (创建的时候，修改名称，检索空间)
capacity   （创建的时候，扩容，作品上传，作品覆盖，作品复制，作品分享等涉及到作品进入空间和作品移出空间的操作）
source       (创建，修改)
limits         (创建，修改，用户加入空间或者用户移出空间)
open         (创建，修改，页面上筛选)
owner       (创建，修改，移出作品）
createTime(创建)
state         （创建，修改。冻结的空间不能被mixer访问，不能够上传作品)
rentable    （创建租赁空间，加入租赁空间的作品都是可租赁的作品，需要到租赁中心注册）
memberNum （创建空间时默认是0，每有一个用户加入空间就+1，每有一个用户退出空间就-1)
worksnum (创建空间时默认是0，每有一个作品加入空间就+1，每有一个作品退出空间就-1  作品分享，作品复制，删除作品，上传作品都会涉及到这个字段的变动)
 shared      (共享空间，用户无需加入就能访问里面的作品，在创建空间时可以指定，mixer中查看用户能访问的作品时需要将共享空间里的作品加上)
packageSupport（是否支持打包，创建空间时可以指定，也可以修改该属性。creator中询问用户可上传的空间时需要摒弃不支持打包的空间）
usedCapacity(空间已用容量，空间创建的时候指定为0，每有一个作品进入空间就加上该作品的大小，每有一个作品被移除就减去这个作品的大小，必须大于0 如果减去后的大小小于0则设置为0)

2 works 

worksID (字符串，作为作品的唯一标识，不可重复，通常用雪花算法生成，一旦生成不可被修改)
worksName (作品名称，在打包作品，作品覆盖，二次创作等作品入库的时候检查名称是否合法)
insertTime(打包时自动生成，不可更改)
lastModifyTime(对作品的属性做一次修改就变更一次)
category (作品类别，现在没啥用，可能需要在入库的时候校验类别是否存在)
label(作品标签，也没啥用）
uploaderID (上传者ID,打包的时候指定，无法更改，用于查询TOP最活跃的用户)
uploaderName 
ownerID (所有者ID ，打包时默认为上传者ID, 可以转让所有权 ，参与查看个人作品逻辑，在用户移出服务器时需要删除owner所拥有的作品)
ownerName
spaceID  (初始ID ,打包时指定，该ID必须能找到一个对应的space ,在空间删除时需要将spaceID下的作品删除，不可修改)
download（打包时默认是0，每下载一次就+1，参与作品下载，按下载量查询作品，下载TOP的运算）
score (打包时默认是0 ，每天的定时任务中需要计算每个作品的评分然后更新)
state (默认是1，参与作品冻结与解冻逻辑， 作品被冻结的时候只有作品所有者能看见，不能被其他用户看见)
size (打包时自带，不可被修改)
description (打包自带，之后可修改)
open (私人和公开作品，暂时没啥用)
version （打包自带，不允许修改）
creationSupport（属于二次创作的作品不支持二次创作，不属于原创的则支持二次创作）
parent (二次创作的作品的父作品ID,打包自带，不可修改)
type(打包自带，不可修改 ,有可能用于区分横屏作品和竖屏作品)
platform(打包自带，可修改 ,有可能按照平台区分作品)
icon(图标地址，相对地址,打包时自带，可以被修改)
address(作品源地址，相对地址,打包时自带，可以被修改)  
hash(源文件的hash值，打包作品计算，覆盖作品的时候重新计算)
iconName (图标文件的文件名,打包自带，作品覆盖时会被修改) 
srcName(源文件的文件名,打包自带，作品覆盖时会被修改)    
viewTime(被浏览了多少分钟，定时任务每天处理一次)     
purchaseNum(被购买次数，打包时是0，用户每购买一次+1)
description(作品描述，可以修改)  
state(默认未冻结。可修改，冻结了的作品只能被所有这所看见） 
collectnum(被收藏的数量，默认是0）
